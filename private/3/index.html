리디북스 DRM PDF 추출 방법

rdport 프로젝트 - 윈도우 기준 device ID 찾기 및 복호화 과정 정리

개요

리디북스 앱은 PDF 책을 AES-128-CBC 2중 암호화로 저장한다.

[.dat 파일] → 복호화 → content key 추출
[.pdf 파일] → content key로 복호화 → 원본 PDF

파일 구조

.dat 파일 (128 bytes)

[0..16]   - IV (16 bytes)
[16..128] - AES 암호화된 데이터 (112 bytes)

복호화하면 나온 결과의 [68..84] 바이트가 content key.

.pdf 파일

[0..16]  - IV (16 bytes)
[16..]   - AES 암호화된 PDF 본문

복호화 키 찾는 방법

핵심

settingsDeviceId().slice(0, 16) 이 .dat 복호화 키다.

즉, 리디북스 앱 내부의 실제 device ID (UUID 형식) 앞 16글자를 그대로 AES 키로 쓴다.

실제 device ID 위치

%APPDATA%\Ridibooks\datastores\global\Settings 파일 안에 있는데, 이 파일 자체가 암호화돼 있다.

Settings 파일 복호화 방법:

  1. 암호화 방식: CryptoJS AES-ECB + NoPadding, 커스텀 PKCS7
  2. 복호화 키: Windows Credential Manager의 com.ridi.books / global 값 (base64 디코딩한 UUID)
  3. 암호화된 데이터 위치: 파일 오프셋 256 바이트 이후

Windows Credential Manager에서 UUID 읽기:

# PowerShell에서 (관리자 불필요)
$cred = [System.Net.NetworkCredential]::new("", (
    [System.Runtime.InteropServices.Marshal]::PtrToStringBSTR(
        [System.Runtime.InteropServices.Marshal]::SecureStringToBSTR(
            (Get-StoredCredential -Target "com.ridi.books/global").Password
        )
    )
)).Password
# 또는 Win32 CredRead API로 읽은 뒤 base64 디코딩

Node.js로 Settings 파일 복호화:

const CryptoJS = require("crypto-js");
const fs = require("fs");

// 1. Credential Manager에서 읽은 UUID를 키로 변환
function makeKey(uuid) {
  const t = CryptoJS.enc.Utf8.parse(uuid);
  if (uuid.length % 16) CryptoJS.pad.Pkcs7.pad(t, 4); // 48 bytes로 패딩
  return t;
}

const credManagerUUID = "<Credential Manager에서 읽은 UUID>";
const key = makeKey(credManagerUUID);

// 2. Settings 파일 복호화 (offset 256부터가 암호화 데이터)
const settings = fs.readFileSync(
  process.env.APPDATA + "/Ridibooks/datastores/global/Settings"
);
const encrypted = settings.subarray(256);
const wordArray = CryptoJS.lib.WordArray.create(Uint8Array.from(encrypted));
const cipherParams = CryptoJS.lib.CipherParams.create({ ciphertext: wordArray });
const decrypted = CryptoJS.AES.decrypt(cipherParams, key, {
  iv: CryptoJS.enc.Hex.parse(""),
  mode: CryptoJS.mode.ECB,
  padding: CryptoJS.pad.NoPadding,
});

// 3. device ID 추출
const json = JSON.parse(CryptoJS.enc.Utf8.stringify(decrypted));
const deviceId = json.data.device.deviceId;
console.log("deviceId:", deviceId);

PDF 복호화 (Node.js)

const crypto = require("crypto");
const fs = require("fs");

// deviceId: Settings 파일에서 추출한 UUID
const deviceId = "<위에서 찾은 device ID>";
const datKey = deviceId.slice(0, 16); // 앞 16글자 그대로 키로 사용

function aes128cbcDecrypt(data, key, iv) {
  const d = crypto.createDecipheriv("aes-128-cbc", key, iv);
  d.setAutoPadding(false);
  return Buffer.concat([d.update(data), d.final()]);
}

const bId = "6071000282"; // 책 ID
const bookDir = `C:/Users/zkzkz/Downloads/리디/${bId}`;

const dat = fs.readFileSync(`${bookDir}/${bId}.dat`);
const pdf = fs.readFileSync(`${bookDir}/${bId}.pdf`);

// 1단계: .dat 복호화 → content key 추출
const decDat = aes128cbcDecrypt(dat.subarray(16), datKey, dat.subarray(0, 16));
const contentKey = decDat.slice(68, 84); // 바이트 [68..84]

// 2단계: .pdf 복호화
const decPdf = aes128cbcDecrypt(pdf.subarray(16), contentKey, pdf.subarray(0, 16));

if (decPdf.slice(0, 4).toString() === "%PDF") {
  fs.writeFileSync(`${bookDir}/${bId}_decrypted.pdf`, decPdf);
  console.log("완료!");
}

책 파일 위치

리디북스 앱이 책을 저장하는 경로:

  • 앱 기본값: %APPDATA%\Ridibooks\library\_12132669\{bId}\
  • 커스텀 설정 시: 앱 설정 > 다운로드 경로에서 확인

주의사항

  • device ID는 앱 재설치 또는 디바이스 재등록 시 바뀔 수 있음
  • 바뀌면 기존 방법대로 Settings 파일 다시 복호화해서 새 UUID 추출
  • Credential Manager UUID도 같이 바뀌므로 세트로 업데이트 필요
